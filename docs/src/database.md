# 数据库

这里用于整理数据库相关面试题，后续按 SQL、索引、事务、锁、MVCC、慢查询优化、分页优化、连接池、Redis 等方向追加。

## 1、数据库索引为什么能提升查询速度
数据库索引是一种辅助数据结构，可以帮助数据库更快定位数据，避免对整张表做全表扫描。常见关系型数据库中，B+ 树索引非常常见。

::: details 详情
### 没有索引时

如果没有索引，查询可能需要逐行扫描：

```sql
select * from users where name = 'Tom';
```

数据量越大，全表扫描成本越高。

### 有索引时

如果 `name` 字段有索引，数据库可以先在索引结构中查找 `Tom` 对应的位置，再回表读取完整数据。

这类似查字典时先看目录，而不是从第一页翻到最后一页。

### B+ 树索引特点

B+ 树适合数据库索引，原因包括：

- 树高较低，减少磁盘 IO。
- 叶子节点有序，适合范围查询。
- 非叶子节点只保存索引键，能容纳更多索引项。
- 查询、插入、删除都有较稳定的复杂度。

### 索引的代价

索引不是越多越好：

- 会占用额外存储空间。
- 插入、更新、删除时需要维护索引。
- 低选择性字段建索引收益可能不高。
- 复杂查询中索引可能不被优化器选择。

### 常见适合建索引的字段

- 经常出现在 `where` 条件中的字段。
- 经常用于 `join` 的字段。
- 经常用于 `order by` 或 `group by` 的字段。
- 高区分度字段，例如手机号、订单号。

### 面试要点

- 索引通过减少扫描范围提升查询性能。
- 索引会降低写入性能，需要权衡。
- 联合索引要注意最左前缀原则。
- 是否使用索引要结合执行计划分析。
:::
